# Repository: agents + skills catalogue

A foundational catalogue of reusable Claude Code subagents and skills, distributed as plain markdown.

## Conventions

- **Subagents:** `.claude/agents/<name>.md`
- **Skills:** `.claude/skills/<skill-name>/SKILL.md`
- **Shared context:** `.claude/shared/` — organizational strategy, personas, and archetypes that agents and skills reference. Read [`shared/README.md`](.claude/shared/README.md) for the full layout.
- Names are kebab-case, lowercase. Filename stem (agents) or directory name (skills) MUST match the `name:` frontmatter field.
- Spec source of truth: [`docs/anthropic-spec.md`](docs/anthropic-spec.md). Project conventions: [`CONTRIBUTING.md`](CONTRIBUTING.md).

## Workflows

| Command | Purpose |
| --- | --- |
| `/new-agent [name]` | Interview-driven scaffolding for a new subagent. |
| `/new-skill [name]` | Interview-driven scaffolding for a new skill. |
| `/migrate-prompt` | Import an existing prompt blob as an agent or skill. |
| `/validate-catalogue [path]` | Lint frontmatter and naming. Whole catalogue if no path. |
| `/index-readme` | Regenerate README catalogue tables from `.claude/` contents. |

**Before committing entries that add or rename agents/skills, run `/validate-catalogue` and `/index-readme`.** The `.githooks/pre-commit` hook (enabled via `scripts/install-hooks.sh`) and the `.github/workflows/validate.yml` CI workflow enforce both automatically.

## Plugin export

`scripts/export-plugin.sh` produces a `.claude-plugin/`-formatted copy under `dist/plugin/` for `/plugin install` distribution. Not part of day-to-day workflow.

## Where things live

```
.
├── CLAUDE.md                ← this file
├── README.md                ← catalogue index (regenerated by /index-readme)
├── CONTRIBUTING.md          ← naming + frontmatter spec, add-an-entry guide
├── docs/anthropic-spec.md   ← canonical spec, last-verified date
├── scripts/                 ← validate.py, index-readme.py, export-plugin.sh
└── .claude/
    ├── agents/              ← one .md per subagent
    ├── skills/              ← one directory per skill
    ├── commands/            ← slash commands powering the workflows above
    └── shared/              ← strategy/persona/archetype context for agents
        ├── company/         ← Acme Computing (identity, strategy, charter)
        ├── organizations/   ← CDG Commercial
        ├── portfolios/      ← Pro Workstation
        ├── products/        ← Pro Workstation PC (focus today)
        ├── personas/        ← internal (agent users) + external (customers)
        └── archetypes/      ← behavioral patterns across personas
```
